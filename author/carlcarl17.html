    svn delete only in repo

    版本管理一些檔案像是測資之類的可能會不小心用 svn add 起來，但是有些測資檔案很大，實在不想上傳，所以就想要將檔案從 svn 版本中移除。

    之前都很笨的先備份檔案，做刪除，然後再複製回來，後來查資料才發現有參數可以用:

    svn delete --keep-local file

    這樣就能保留這個檔案了～。

    參考網頁:
    http://www.lampblog.net/2011/01/svn-delete

    awstats analyze large logs at once (include gz files)

    awstats的簡易安裝教學

    因為在維護的網站log檔有一年份，而且都已經壓縮成gz檔，要用awstats分析的話實在很費工夫，後來發現awstats有好用的工具可以用：logresolvemerge.pl。

    簡單講就是用這個工具將log檔(包含gz壓縮檔)都merge起來為一個大檔案後，再用awstats作分析。

    這邊首先要注意的就是 awstats不會讀較舊的log檔，所以像我之閒已經讀了最新的log檔後想讀舊的，就只能直接砍掉紀錄檔重新開始，到 /var/lib/awstats/底下將底下的檔案刪光，如此就能回到最初的狀態

    logresolvemerge.pl預設是在/usr/share/doc/awstats/examples/裡
    可以移到其他地方，不然每次打路徑應該會打到手很酸XD

    這裡給個幾個簡單的範例

    將多個log檔merge成一個log檔

    logresolvemerge.pl /var/log/apache2/.log > temp.log

    將多個gz檔merge成一個log檔(其實跟上面的差不多~)

    logresolvemerge.pl /var/log/apache2/.gz > temp ...

    vim xptemplate add smarty(tpl) support

    弄了很久 查了一下資料才發現原來那麼簡單=.=。

    在 personal_ftplugin/ 底下要先建一個 smarty 資料夾，接著在裡面新增一個 smarty.xpt.vim 檔案。

    以下是檔案的內容:

    XPTemplate priority=personal
XPTinclude
html/html

    這樣就 ok 了，其實就是把 html 的設定拿來用而已。接著開一個 tpl 副檔名的試試看，應該是 ok。我想應該底層有去做 smarty 和 tpl 副檔名的對應，要不然這邊的內容完全都沒看到 tpl 的對應。
